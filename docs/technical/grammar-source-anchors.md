# Grammar Source Anchors

This document names the source-backed grammar works LangNet should use when it
maps Foster functional grammar into traditional Greek, Latin, and Sanskrit
terms. These anchors are bibliography and discovery targets first. A work-level
anchor is not enough to prove a specific concept; concept evidence should point
to a verified reader segment when we claim that a passage supports a teaching
rule.

## Evidence Rule

Use two evidence levels:

- `reader_work`: a canonical source work that should be shown as a grammar
  reference or used for source discovery.
- `reader_segment`: a specific passage in that work that supports a concept,
  term, process, or example.

Only `reader_segment` evidence should back wording such as "this source says
that the genitive means possession." Work-level anchors can back wording such
as "for Greek grammar, start with Dionysius Thrax and Apollonius Dyscolus."

External catalog and encyclopedia pages below are identity and bibliography
checks. They help confirm that the work is a real canonical or major grammar
source, but the actual teaching claim should still be grounded in our reader
text when possible.

## Concept Evidence Shape

The grammar concept registry should grow from compact `source_basis` strings
into structured evidence links:

```yaml
evidence:
  - evidence_level: reader_work
    source_anchor_id: grammar.source.panini.astadhyayi
    work_id: langnet:reader:sanskrit_dcs:dcs_413
    canonical_text_id: urn:ctsv2:san:astadhyayi-vrddhir-adaic
    label: "Pāṇini, Aṣṭādhyāyī"
  - evidence_level: reader_segment
    source_anchor_id: grammar.source.panini.astadhyayi
    work_id: langnet:reader:sanskrit_dcs:dcs_413
    canonical_text_id: urn:ctsv2:san:astadhyayi-vrddhir-adaic
    citation_path: "550729"
    canonical_address: urn:ctsv2:san:astadhyayi-vrddhir-adaic?ref=550729
    label: "Pāṇini, Aṣṭādhyāyī: adeṅ guṇaḥ"
```

Use `reader_work` for bibliographies, tooltips, and source discovery. Use
`reader_segment` for individual concept assertions, examples, and lesson text.

## Discovery Commands

Use these commands to refresh or audit the anchor list:

```bash
just cli reader works --language lat --group grammar --sort group-popularity --limit 20 --output json
just cli reader works --language grc --group grammar --sort group-popularity --limit 20 --output json
just cli reader works --language san --group grammar --sort group-popularity --limit 20 --output json
```

For segment-level research:

```bash
just cli reader contents langnet:reader:sanskrit_dcs:dcs_413 --limit 20 --output json
just cli reader show langnet:reader:sanskrit_dcs:dcs_413 --segment 550729 --output json
```

The same pattern applies to the Greek and Latin anchors once the specific
reader citation is known.

## Actionable Use In The Learning Overlay

1. Add `source_anchor_id` values for the table above. **Done for the initial
   registry slice.**
2. Extend `GrammarConcept` with optional structured `evidence`. **Done for the
   initial registry slice.**
3. Keep existing `source_basis` strings as fallback prose until every core
   concept has at least one structured work anchor.
4. Promote a source from `reader_work` to `reader_segment` only after checking
   the passage in the local reader.
5. Surface work-level anchors in CLI/web as "source tradition" links, and
   segment-level anchors as "evidence for this concept" links.
